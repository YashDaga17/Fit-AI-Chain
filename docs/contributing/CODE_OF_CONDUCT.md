# Code of Conduct

Fit AI Chain is committed to maintaining a respectful and collaborative community environment.

---

# Our Goals

We aim to create a community that is:

- Respectful
- Inclusive
- Collaborative
- Beginner-friendly
- Professional

The project encourages positive collaboration and learning-focused contribution experiences.

---

# Expected Behavior

Contributors are encouraged to:

- Be respectful to others
- Communicate clearly
- Provide constructive feedback
- Support new contributors
- Collaborate professionally

---

# Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment
- Personal attacks
- Discrimination
- Trolling
- Intentionally disruptive behavior

---

# Collaboration Standards

Contributors should:

- Respect project maintainers
- Respect review feedback
- Keep discussions constructive
- Avoid unnecessary conflicts

Constructive feedback should focus on improving the project rather than criticizing contributors personally.

---

# Issue and Pull Request Etiquette

When participating in issues or pull requests:

- Stay on topic
- Provide useful information
- Avoid spam or duplicate discussions
- Communicate respectfully

---

# Beginner-Friendly Environment

The project welcomes:

- First-time contributors
- Documentation contributors
- Open-source beginners
- Students and learners

Questions and learning-oriented discussions are welcomed within the community.

---

# Reporting Problems

If you encounter inappropriate behavior:

- Open a GitHub issue or discussion
- Contact project maintainers through the repository
- Provide clear information
- Remain respectful during reporting

---

# Enforcement

Project maintainers review reported violations on a case-by-case basis.

Possible actions may include:

- Warning contributors
- Temporary restriction from discussions or contributions
- Permanent removal from the community for severe violations

Maintainers aim to handle reports respectfully and fairly while protecting the community environment.

Contributors may contact maintainers through repository discussions if they believe an enforcement action was applied incorrectly.
