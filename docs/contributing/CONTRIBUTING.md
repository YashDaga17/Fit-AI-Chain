# Contributing Guide

Thank you for contributing to Fit AI Chain.

This document explains how contributors can participate in the project.

---

# Contribution Goals

Contributors can help with:

- Documentation improvements
- Bug fixes
- UI enhancements
- API improvements
- Performance optimization
- Feature development

---

# Before Contributing

Before starting:

- Read project documentation
- Check existing issues
- Avoid duplicate work
- Discuss large changes when necessary

Contributors are encouraged to comment on issues before starting major work.

---

# Recommended Workflow

```text
Fork Repository
  ↓
Clone Repository
  ↓
Create Branch
  ↓
Make Changes
  ↓
Test Changes
  ↓
Commit Changes
  ↓
Push Branch
  ↓
Open Pull Request
```

---

# Setup Instructions

Clone the repository:

```bash
git clone https://github.com/your-username/Fit-AI-Chain.git
cd Fit-AI-Chain
```

Install dependencies:

```bash
npm install
```

Start development server:

```bash
npm run dev
```

---

# Branch Naming

Recommended branch naming:

| Type | Example |
|---|---|
| Documentation | `docs/api-improvements` |
| Feature | `feature/meal-suggestions` |
| Fix | `fix/auth-bug` |

---

# Pull Request Guidelines

Pull requests should:

- Stay focused on one topic
- Include clear descriptions
- Reference related issues
- Avoid unrelated changes

Reference the related GitHub issue number inside pull request descriptions.

---

# Documentation Contributions

Documentation updates should:

- Use consistent markdown formatting
- Remain beginner-friendly
- Maintain clean structure
- Avoid unnecessary duplication

Documentation structure consistency is important for long-term project maintainability.

---

# Code Contributions

Code contributions should:

- Follow project standards
- Maintain readability
- Avoid duplicated logic
- Keep components modular

---

# Testing Before Submission

Before submitting:

- Run the application locally
- Verify changes work correctly
- Check for console errors
- Ensure documentation is updated

---

# Communication Guidelines

Contributors should:

- Be respectful
- Communicate clearly
- Respond to feedback constructively
- Collaborate professionally
