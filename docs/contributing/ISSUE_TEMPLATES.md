# Issue Templates

This document explains recommended issue formats for Fit AI Chain.

---

# Why Use Issue Templates?

Issue templates help:

- Improve clarity
- Reduce confusion
- Simplify contributor collaboration
- Make debugging easier
- Organize feature discussions

Structured issues help maintainers review and prioritize work more efficiently.

---

# Bug Report Template

Recommended structure:

```md
## Bug Description

Describe the issue clearly.

---

## Steps to Reproduce

1. Open the application
2. Navigate to the affected page
3. Perform the action
4. Observe the issue

---

## Expected Behavior

Describe what should happen.

---

## Actual Behavior

Describe what actually happens.

---

## Screenshots

Add screenshots if applicable.

---

## Environment

- OS:
- Browser:
- Device:
```

---

# Feature Request Template

Recommended structure:

```md
## Feature Summary

Describe the feature clearly.

---

## Problem Statement

Explain the problem this feature solves.

---

## Proposed Solution

Describe the proposed implementation.

---

## Additional Context

Add screenshots, references, or examples.
```

---

# Documentation Issue Template

Recommended structure:

```md
## Documentation Area

Specify affected documentation.

---

## Problem

Describe what is unclear or missing.

---

## Suggested Improvement

Describe the recommended update.
```

---

# Good Issue Practices

Good issues should:

- Be specific
- Include reproduction steps
- Avoid duplicate reports
- Stay focused on one topic

Clear issue descriptions help contributors avoid duplicate or conflicting work.

---

# Before Opening an Issue

Check:

- Existing issues
- Existing pull requests
- Documentation pages

Contributors are encouraged to discuss larger implementation ideas before beginning work.
